# 📦 نظام MSP لتوثيق زيارات فريق المبيعات
### MSP Sales Team Visit Documentation System (V3.0)

نظام تقني متكامل وموحد مخصص لتوثيق وتحليل الزيارات الميدانية التي يقوم بها فريق المبيعات في شركة **Modern Style Pack (MSP)**. يهدف النظام إلى رفع كفاءة التوثيق، ضمان دقة البيانات الميدانية، وتحقيق المطابقة المالية والتحليلية الفورية عبر لوحات قيادة تفاعلية.

---

## 🚀 الأهداف الرئيسية للنظام (Core Objectives)
* **توثيق الزيارات:** تسجيل لحظي لكل زيارة ميدانية مع التقاط تلقائي للموقع الجغرافي (Geolocation).
* **دقة البيانات:** الربط الديناميكي مع جداول المناطق، المحافظات، والقطاعات المعتمدة في قاعدة البيانات.
* **التحليل الإحصائي (Analytics):** لوحة تحكم تفاعلية متكاملة (Dashboard) تستخدم تقنيات **Chart.js** لتحليل الأداء وتسلسل الزيارات زمنياً وجغرافياً.
* **الرقابة والشفافية:** نظام سجلات (t98_logs) لرصد نشاط الفريق وتوقيت العمليات بشكل دقيق وخفي.
* **الهوية الموحدة:** واجهة مستخدم احترافية (Premium Glassmorphism) تعكس هوية MSP الحديثة (Emerald Green `#10b981`).

---

## 🛑 معايير التطوير الصارمة (23 MSP Project Standards)
تم رفع كفاءة هذا النظام وبنائه بناءً على دستور معايير MSP، والذي يتضمن:
1. **Three-File Protocol**: فصل تام للمنطق والتصميم والهيكل (HTML, CSS, JS) لكل صفحة.
2. **Global Config Injection**: الاعتماد على كود مركزي لبناء الهيدر، والقائمة الجانبية (Offcanvas Sidebar)، وتوحيد الهوية البصرية.
3. **Bilingual Labels & Comments**: كتابة الكود والتعليقات وعناصر الواجهة باللغتين (العربية والإنجليزية).
4. **Non-blocking UX**: تبني نظام الإشعارات (Toast Notifications) بدلاً من النوافذ المنبثقة المزعجة لتجربة مستخدم سلسة.
5. **Data Protection**: ربط عمليات الحذف بنوافذ تحقق (Confirmation Modals) لحماية البيانات من الحذف العرضي.

---

## 🛠️ الهيكلية البرمجية التقنية (System Architecture)
يعتمد النظام على بنية **"المحرك الموحد"** لضمان سهولة الصيانة وتوحيد الواجهات:

* **القاعدة الذهبية لتسمية قواعد البيانات (Golden Rules):**
    * جميع الجداول تبدأ بحرف `t` (مثل: `t01_visits`).
    * جميع الحقول تبدأ بحرف `f` (مثل: `f17_visit_date`).
* **التقنيات المستخدمة:** 
    * **Frontend:** HTML5, Vanilla CSS3 (Grid/Flexbox/Glassmorphism), Vanilla JavaScript.
    * **Backend/DB:** Supabase (PostgreSQL, REST API).
    * **Libraries:** Chart.js (وثائق التحليل), FontAwesome (أيقونات النظام).

## 📂 خريطة ملفات المشروع الرئيسية (File Mapping)
يلتزم النظام باتفاقية تسمية صارمة:

```text
/msp-sales-system
│
├── global-config.js      # "المحرك الذكي": بناء هيدر تفاعلي، القائمة الجانبية، والاتصال الموحد بقاعدة البيانات.
│
├── visits.html           # واجهة المستخدم لتوثيق الزيارات الميدانية وعرض الجداول المتفاعلة.
├── visits.css            # التصميم الخاص بالنماذج والجداول.
├── visits.js             # معالجة إرسال البيانات إلى `t01_visits` وتسجيل السجلات وتتبع الموقع.
│
├── reports.html          # اللوحة التحليلية والرسوم البيانية لتوجهات الزيارات (Analytics Dashboard).
├── reports.css           # تصميم شبكة الـ (Cards) ومؤشرات الأداء.
└── reports.js            # سحب البيانات من السيرفر وبناء مخططات Chart.js المتقدمة.
```
